Description — EV-01 기부사업 (PC)

#영역설명 · 배치 의도
1헤더확정GNB ‘이벤트’→‘기부사업’ + ‘지정기부’ 탭 제거(일원화 결정 2026-06-27). 라우트 /event/eventList 재활용 또는 신규.
2페이지 히어로확정 카피천원시리즈 스토리텔링 도입부(웰로 대비 차별 콘텐츠 — 6/10 미팅 요구⑤·본부장 요청). 공 과장 톤.
3성과 카운터BO‘규모’를 숫자로 전달(누적 기부자·기부금·사업·수혜). 실수치 소스 확정 필요(6/29).
4★지금 모금 중 (지정기부)확정e음 API 출력 섹션(statusCd=2만·우리 BO 수정 불가). 현재 광주서구 진행 중 지정기부 0건 → 빈 상태가 실화면(2건은 종료·아래는 모금 개시 시 예시). 카드 클릭→모금 상세(SG-02·[기부하기]). 기저귀(8월)·천원세탁 재모금 시 노출. 지정기부를 별도 페이지 대신 이 섹션으로 일원화한 핵심 영역.
5진행 중 기부사업 (이야기)확정BO 콘텐츠·천원시리즈 한정(김희선 주무관·2026-06-27). 집행형(국시·택시)=성과/지정기부형(세탁·기저귀)=소개+위 모금섹션 연결. 메인 ⑦과 동일 데이터. 📌 등록 주체 = Phase1 운영자(우리) 대행 / Phase2 지자체(광주서구) 직접 BO 등록(2026-06-29 확정).
6곧 시작협의예정 사업 노출로 ‘적은 아이템’ 보강(6/29 노출 OK 확인). 기저귀=모금/사업 시점 분리.
7영상·이야기BO천원국시 유튜브 3편(7호점·뉴스·10호점). 메인 소식 섹션과 자산 공유.
8하단 유도확정답례품·지정기부(위 모금섹션)로 행동 전환.
9푸터확정메인과 동일.
📌 일원화 구조(2026-06-27 결정): 지정기부 별도 탭 없이 기부사업 1페이지 = [모금 섹션 e음] + [이야기 섹션 BO]. 사유=진행 중 지정기부 모금 0건(2건은 종료)이라 별도 페이지 텅 빔. 근거 [[20260627] 기부사업 vs 지정기부 IA] · 기획안 [20260627_1730]. 6/29 협의 후 확정.
